# Ductus docs website (Journey)

This directory is an Astro website generated by Ductus (without Starlight); the data in `ductus.data.json` is generated by Ductus. Install the dependencies once with `npm install`. After that, `npm run dev` starts a local preview and `npm run build` produces the static website under `dist/`.

## Structure

- **`ductus.data.json`** — the complete data contract (dataVersion 1): site
  metadata, journeys with nodes, edges, main path, LLM markdown and
  faithfulness warnings. Ductus overwrites the file on every `ductus generate`;
  the bundled version contains demo data for the local preview. If the file is
  missing, the site still builds with an empty fallback (defensive, see
  `src/lib/data.ts`).
- **`src/pages/index.astro`** — overview: search, “Frequently searched” chips
  (most frequent edge labels), journey cards, footer.
- **`src/pages/journeys/[slug].astro`** — journey detail: interactive graph
  (deterministic BFS layout, Bezier edges), step list from the main path,
  “Further actions”, faithfulness banner and the “Detailed guide”
  (LLM markdown, rendered safely at build time).
- **`src/lib/`** — data loading, step derivation, graph layout, search index
  and central UI strings (English as the default; German when `site.locale`
  starts with `de`).

## Interaction

- **⌘K / Ctrl+K** opens the search (journeys, steps, decisions, actions);
  arrow keys + Enter navigate, `esc` closes.
- **Clicking a node** jumps to the matching step on the right; **“Play path”**
  steps through the main path with animation. `prefers-reduced-motion` is
  respected (animations off, scrolling without smooth behavior).
- The Google fonts (Instrument Sans, Spline Sans Mono) are loaded via CDN;
  without an internet connection the `system-ui` fallback kicks in — the page
  remains fully usable.

Please make content changes in the app's source code (annotations), not in the
generated data.
